# Phase 0 UX Package

This package locks the product-level UX decisions that must be stable before any
screen redesign or implementation work begins.

Contents:
- `architecture.md`: information architecture, shell, screen roles, mode
  boundaries, and interaction rules
- `terminology.md`: naming system, plain-language mapping, and blocker/error
  copy rules
- `journeys.md`: canonical user journeys and screen transitions

Phase 0 goals:
- make the ordered document stack the visible product model
- define a non-technical default shell without removing advanced control
- separate top-level work, shared terms, and setup from document-level tools
- lock copy and screen roles before visual design starts

Phase 0 is complete when:
- the top-level IA is accepted
- the complexity-control model is accepted
- the terminology matrix is accepted
- the journeys in `journeys.md` are accepted as canonical

Out of scope for Phase 0:
- visual styling
- component sizing and spacing
- implementation details
- migration sequencing
