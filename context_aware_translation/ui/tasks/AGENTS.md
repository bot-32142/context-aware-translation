# tasks

## Purpose
Qt integration layer for the workflow task engine. Bridges pure-Python task execution with Qt signals/slots, provides task state mapping to UI view models, and displays task logs in a console widget.

## Key Files
| File | Description |
|------|-------------|
| `qt_task_engine.py` | QObject wrapper around EngineCore. Emits signals for task changes, status messages, and errors. Manages task lifecycle (submit, start, cancel). |
| `task_console.py` | Task log console widget. Displays colorized task output and updates in real-time. |
| `task_view_model_mapper.py` | Pure mapping function: TaskRecord → TaskRowVM. Normalizes progress fields, generates display titles and scope labels. |
| `task_view_models.py` | Data classes for UI representation (TaskRowVM, etc). Not responsible for database access. |

## For AI Agents
### Working In This Directory

**TaskEngine (qt_task_engine.py):**
- Wraps EngineCore to add Qt signals
- Signals: `tasks_changed(book_id)`, `status_message(style, message)`, `error_occurred(message)`, `running_work_changed(is_running)`
- Public API mirrors EngineCore: `submit()`, `preflight()`, `get_tasks()`, `cancel_task()`
- Uses QTimer for autorun polling
- All mutations emit appropriate signals for connected UI views

**Task State Flow:**
1. `submit(task_type, book_id, **params)` - Create task and attempt to run
2. Engine transitions through states: pending → running → completed|failed|cancelled
3. Signals notify views of changes
4. Views call `get_tasks()` to refresh display
5. `cancel_task()` stops running work and marks task cancelled

**TaskRowVM (task_view_models.py):**
- Display model for a single task row
- Fields: task_id, book_id, task_type, title, scope_label, status, phase, progress counters, error, timestamps
- Immutable after construction
- Generated by `map_task_to_row_vm()`

**Mapping Function (task_view_model_mapper.py):**
- `map_task_to_row_vm(record: TaskRecord) -> TaskRowVM`
- Pure function, no side effects
- Normalizes progress fields to non-negative ints (handles null/invalid gracefully)
- Generates display titles from task type constants
- Computes scope label (all docs, specific doc count, or no scope)
- Task types with no document scope: glossary_export, glossary_review, glossary_translation

**TaskConsole (task_console.py):**
- QWidget with QPlainTextEdit or similar for log display
- Colorizes task output (info, warning, error levels)
- Updates in real-time as worker emits progress/log signals
- Optional: search/filter by log level

### Common Gotchas
- Progress normalization: invalid values (None, negative, non-numeric) become 0
- Task type strings must match entries in `_TASK_TYPE_TITLES` dict
- `scope_label` computed from `document_ids_json` field (may be unparseable; defaults to "All documents")
- TaskEngine signals use `QueuedConnection` to avoid cross-thread issues
- `get_tasks()` returns empty list if book has no tasks
