# 上下文感知翻译（CAT）

CAT 是一个桌面翻译工具，面向长文本与多种文档（书籍、PDF、EPUB、漫画、扫描页），强调术语一致性与跨文档上下文。

## 为什么用 CAT

- 自动从语料中抽取术语以及相关描述并构建可用术语表。
- 每个术语按导入顺序累积上下文，在翻译时插入相关术语的相关描述。
- 保证相关术语译名一致。
- 在支持的文档类型中尽量保留原结构/版式（例如 EPUB 结构保留、文本目录结构导出）。
- 一套流程覆盖文本、EPUB、PDF、扫描图像、漫画。
- 支持 OCR 与人工复核后再导出。
- 支持图片嵌字

## 核心思路：上下文管理

CAT 不把每个分块当成孤立文本翻译。

1. 按阅读顺序导入并切分文档。
2. 自动抽取术语并建立出现位置映射。
3. 用上下文树持续汇总术语描述。
4. 翻译时仅为当前批次注入相关术语（术语名 + 术语翻译 + 汇总描述）。

结果是：在不每次发送整本上下文的前提下，获得更好的长文一致性。

## 推荐流程

1. `导入`：按预期阅读顺序导入文档。
2. `OCR审核`（如需要）：先跑 OCR，再修正 OCR 文本。
3. `术语表`：构建/导入术语表，复核并翻译术语。
4. `翻译`：翻译选中文档。建议每次发送少量内容以获得稳定高质量输出。过多的上下文必然导致幻觉。建议控制在3000-4000令牌以内，具体取决于你使用的模型以及是否启用了上下文摘要。
5. `导出`：按文档类型与格式导出。

## 支持类型

| 文档类型 | 导入 | 导出 | 翻译前是否必须完成 OCR |
|---|---|---|---|
| 文本 | 单个或文件夹：`.txt` / `.md` | `txt` | 否 |
| PDF | 单个 `.pdf` | `epub`, `md` | 是 |
| 扫描书籍 | 图片文件/文件夹 | `epub`, `md` | 是 |
| 漫画 | `.cbz` 或图片文件夹 | `cbz` | 是 |
| EPUB | 单个 `.epub` | `epub`, `md`, `docx`, `html` | 否（但支持图片 OCR） |

## llm端点配置推荐

你可以为不同步骤绑定不同端点/模型，以平衡成本、速度与质量。

| 步骤 | 作用 | 端点/模型建议 |
|---|---|---|
| `术语抽取` | 从分块文本中提取术语候选与初始描述 | 该步骤会处理大量输入，优先选低成本、支持输入缓存、结构化输出稳定的模型（如支持 prompt cache 的模型）。强烈建议使用DeepSeek。 |
| `术语描述汇总` | 汇总术语在不同上下文中的描述，构建可复用语义 | 与术语抽取类似，输入量大且重复度高，优先选低成本 + 强输入缓存 + 稳定长上下文处理能力。强烈建议使用DeepSeek。 |
| `术语翻译` | 将术语统一翻译到目标语言 | 优先选术语翻译准确、格式稳定的模型，成本通常低于正文翻译但要保证一致性。推荐训练数据量覆盖面较广，比较便宜的模型如Gemini 3 flash。 |
| `正文翻译` | 执行主文档翻译 | 这是质量最敏感步骤，优先选翻译质量高、风格稳定、长文本表现好的模型。 强烈推荐Gemini 2.5 pro low thinking。Gemini 3或其它强推理模型效果翻译腔往往很重。|
| `文字识别（OCR）`（可选） | 识别图片中的文字并写入可编辑文本 | 需要视觉模型（多模态），优先选 OCR 准确率高、对混排/竖排/噪声图适应好的模型， 如Gemini 3 flash。 |
| `术语审校`（可选） | 对术语进行自动审校（质量检查） | 适合用推理能力较强、指令遵循稳定的模型；可与正文翻译分离以控成本。 |
| `漫画翻译`（可选） | 漫画页级翻译（结合图像语境） | 需要强视觉理解与对话语气处理能力的多模态模型。 |
| `图片嵌字`（可选） | 将译文重新嵌入图片 | 需要支持图片编辑/重绘的后端，优先选版面保留效果稳定的模型。 |

建议实践：

1. 用一个"低成本缓存模型"跑 `术语抽取` + `术语描述汇总`。
2. 用一个"高质量翻译模型"跑 `正文翻译`（必要时也可用于 `术语翻译`）。
3. 视觉相关步骤（`文字识别（OCR）` / `漫画翻译` / `图片嵌字`）单独绑定多模态端点，避免互相影响配额与性能。

以上推荐配置已预设在默认端点和配置方案中（`图片嵌字`除外，需另行配置）。只需填入 API 密钥即可开始使用。

## 注意事项
* OCR无法处理过于复杂的排版。OCR的目的并非保留原格式，而是基于原文内容重新构建原格式。
* 如果原文本单个句子被划归多个自然段，格式有概率可能会乱，也可能会导致翻译失败。
* 所有LLM响应都会尽早缓存并持久化，因此取消操作不会导致数据丢失，你可以随时停止并从中断处继续。
* 图片嵌字（gemini-3-pro-image-preview)针对文字多的图片表现较差。针对漫画，仅有有文字的区域会被分区进行嵌字，所以除了少量ocr无法识别的文字，即便是复杂的漫画质量基本可以保证。普通图片嵌字基于成本考虑和需求考虑目前未做此优化。
